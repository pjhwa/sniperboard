2 아키텍처 전체 구조

두 리포는 “공유 스토리지(GitHub raw JSON)”로 느슨하게 결합된 3계층 파이프라인이다.

Layer 1 — Collect

  • collect_sentiment.py
  • collect_brief.py
  • collect_earnings.py
  • collect_macro_insight.py
  • collect_morning_briefing.py
  • collect_prediction.py (Kalshi)
  • price_context / grok_utils / git_utils
  • monitor/health_check.py
→

Layer 2 — Storage

  • sentiment/latest + history/
  • brief/, earnings/, macro/
  • briefing/, prediction/
  • schema.json (v2.0, 부분 정합)
  • git commit + push
→

Layer 3 — Consume

  • sentiment/brief/earnings services
  • macro_insight / morning_briefing
  • signal_engine · conviction
  • FastAPI /api/*
  • Next.js boards + charts
┌─────────────────────┐ git push ┌──────────────────────┐ raw fetch ┌─────────────────────┐ │ Cron + Hermes/Grok │ ────────────▶ │ GitHub JSON repo │ ────────────▶ │ SniperBoard API │ │ yfinance / Kalshi │ │ latest + history │ │ + Next.js boards │ └─────────────────────┘ └──────────────────────┘ └─────────────────────┘ │ │ │ │ price_context (방향 제거) │ schema_version 2.0 │ composite → conviction │ divergence (사후 계산) │ _en / _ko bilingual │ (현재 스케일 버그)

설계 장점 (사실)

  • 장애 격리: SniperBoard 다운 ≠ 수집 중단
  • 재사용성: raw JSON이면 어떤 소비자도 사용 가능
  • 이력: 추세 차트는 히스토리 파일에서 재구성
  • 비용/지연 분리: LLM은 cron, API는 즉시 응답

5.2 P0 — Conviction 스케일 불일치 (정량 증거)

생산자 composite_score ∈ [-2, +2]. 소비자 calculate_conviction은 max(0, min(100, sentiment))만 수행.

조건: Stage2=5, Regime=60, MIXED

입력 composite   →  정규화된 sentiment(현재)  →  Conviction  →  정상 매핑 시
+0.7 (낙관 약)   →  0.7                       →  46.8 (Low)  →  66.8 (High)
-1.3 (공포)      →  0.0                       →  46.6        →  51.8
+1.8 (과열)      →  1.8                       →  47.1        →  75.1
없음 fallback 50 →  50.0                      →  61.6        →  61.6
효과 심리가 있을수록 점수가 바닥권으로 수렴하고, 데이터 결측 시 오히려 더 좋게 나온다. 리더보드·Watchlist·Daily의 Conviction 배지가 체계적으로 왜곡된다.

13 최종 판정

현재 시스템은 “수집은 정확에 가깝고, 소비·표시는 왜곡” 상태다. Grok 방화벽과 composite 공식, 최신 22심볼 스냅샷은 신뢰할 수 있다. 그러나 Conviction 스케일 오류로 대시보드의 핵심 확신 지표가 체계적으로 틀리며, 심리 추이 차트는 폭 맞춤 결함으로 분석 UX를 해친다. Prediction 파이프라인은 사실상 가동 중이 아니다.
