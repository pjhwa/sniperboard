9.1 잔여 P0 — 즉시

P0-4

Prediction(Kalshi) next_fomc null 복구 + SniperBoard 연동

문제

prediction/latest.json의 next_fomc가 장기간 null. 수집기는 성공 종료. SniperBoard에 서비스/엔드포인트 없음. FOMC 확률 제품이 사실상 다운.

구현 방법
  • Kalshi API 라이브 프로브 (auth, series ticker, 날짜 필드, status 필터)
  • N회 연속 null 시 cron non-zero + health 알림
  • prediction_service.py + GET /api/prediction
  • Macro/Overview 카드 UI
  • 단위 테스트 + 실 API mock 회귀
기대 효과

매크로 의사결정에 시장 함의 확률 제공. 무음 장애 제거. 감사 “고아 파이프라인” 해소.

P0-6

Earnings revenue 단위/통화 가드 (TSM 1263B 등)

문제

yfinance revenue를 무조건 /1e9 → TSM 등 통화·스케일 다른 심볼에서 비현실 매출. Grok 실적 해석 오염.

구현 방법
  • 통화 필드 확인 후 USD 환산 또는 원단위 감지
  • 분기 매출 상한 클램프 (예: 단일 분기 > $200B → null/재시도)
  • collector unit test + 샘플 fixture (TSM/TSLA)
  • 프론트/스키마에 “estimate 신뢰도” 플래그 선택
기대 효과

Earnings Intelligence 숫자 신뢰 회복. AI action_note 환각 감소.
