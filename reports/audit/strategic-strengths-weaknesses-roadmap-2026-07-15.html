3. 데이터 워크플로 (Collect → Store → Serve → Track)

E2E 검증이 커버하는 실제 경로. 유료 Grok 재수집 없이 저장된 JSON + 라이브 yfinance/시그널 경로로 검증.

Collect
market-sentiment-data cron → sentiment / brief / earnings / macro JSON
02
Store
GitHub raw push · schema v2.0 _en/_ko
03
Consume
SniperBoard services + LastGoodCache · serve-time days_until
04
Signal
yfinance · Stage2 · regime · scan_and_log (≥5)
05
Track
outcomes · live stats · vs backtest · health

오염 방화벽 (Contamination Firewall)

가격 방향은 Grok 프롬프트에 절대 들어가지 않는다. 허용: 변동 크기, 거래량 비율, 키레벨 위치. 위반 시 심리 데이터는 가격 에코가 되어 분석 가치가 사라진다.

7. 약점 · 잔여 리스크

고치지 않으면 신뢰가 깎이거나 오용으로 손실이 나는 지점. C1/C2로 설명은 개선됐으나 표본 깊이는 여전히 약함.

✗ AI 해석층 본질 한계

  • 지정학·촉매는 뉴스 2차 요약 — 1차 공시 감사 아님
  • 수치 바인딩 밖 환각 가능

✗ 라이브 청산 n 부족

  • Stage2≥5 + 야간 스캔 → 신규 로그 희소
  • n<30이면 expectancy 해석 보류 (honest) — 엣지 입증 전

✗ 벤더 의존

  • yfinance rate-limit / 조정가
  • 실적 캘린더·EPS 개정 지연

✗ 제품 오용 리스크

  • Conviction High = 매수 신호로 오해
  • Polymarket / 소셜 optimistic 과신

의도적으로 “약점”이 아닌 것

  • AMZN 등 개별 종목 낮은 백테스트 expectancy — 한계 공개가 신뢰 요인
  • reference-only 예측시장 — 과신 방지
  • SPCX 404 — 가짜 Stage2 미표시
  • 라이브 expectancy null (n=0) — 가공 거부

11. 결론

제품 판정

SniperBoard는 2026-07-15 기준 투자 참고용으로 사용 가능한 수준(~93/100)에 도달했다. 기계 시세·Stage2·실적 절대일·배포 계약·브리핑 integrity·Track 방법론/백테스트 대조가 닫혔다. 남은 리스크는 라이브 청산 표본 축적, 벤더 다중화(C3), 예측시장 dual-source(C5)이다. C4 알림·P2+ 출처 감사 링크는 2026-07-16 완료. 매수 버튼이 아니라 반박 가능한 숫자를 앞에 두는 스캐너로 쓰는 한, 설계 의도와 일치한다.
