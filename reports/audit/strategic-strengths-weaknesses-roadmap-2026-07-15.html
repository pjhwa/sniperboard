4. 강점 — 투자 참고 대시보드로서

경쟁 제품 대비 SniperBoard가 실제로 강한 축.

✓ 방법론 기반 시그널

  • Livermore / O'Neil / Minervini 계열 Stage2 체크리스트가 코드로 고정
  • 레짐 5요인 점수 — AI 없이 재현 가능
  • R:R 진입·손절·목표가 기계 산출

✓ 수치 앵커 신뢰

  • 브리핑 가격 테이블 바인딩 (검증: 22/22 exact)
  • 실적 절대일 yfinance 일치
  • VIX·10Y·DXY 등 매크로 레벨 교차검증 가능

✓ 오염 방지 설계

  • 센티먼트 수집: 가격 방향을 Grok에 넣지 않음 (contamination firewall)
  • Polymarket: reference_only — Conviction 미투입
  • 매출 이상치 가드 (P0-6)로 AI 입력 오염 차단

✓ 이중 레이어 아키텍처

  • Collect → GitHub JSON → Consumer 분리
  • serve-time 재계산으로 “어제 쓴 N일 후” 교정
  • 모닝메일·대시보드·API가 동일 정합성 모듈 공유

✓ 운영 가시성

  • 2시간 헬스 모니터 · freshness meta 뱃지
  • 스키마 계약(v2.0 _en/_ko)
  • 감사 HTML 누적 → 의사결정 추적 가능

✓ 제품 표면

  • 보드 분리(Overview/DeepDive/Macro/Sentiment/Briefing…)
  • 한·영 BiLang · 모바일 레이아웃
  • 모닝메일 라이트 에디토리얼 포맷

투자 참고로서의 포지셔닝

SniperBoard는 “추천 종목 앱”이 아니라 규칙 기반 셋업 스캐너 + 검증 가능한 시장 컨텍스트 + (교정된) AI 요약이다. 강점은 신호를 설명 가능하게 만들고, 사용자가 반박할 수 있는 숫자를 앞에 둔다는 점이다.

5. 약점 · 잔여 리스크

고치지 않으면 신뢰가 다시 깎이거나, 오용으로 손실이 나는 지점.

✗ AI 해석층 본질 한계

  • 지정학·촉매 문장은 뉴스 2차 요약 — 1차 공시 감사 아님
  • source_hint는 있으나 원문 링크·인용 깊이는 약함
  • 모델 환각 가능성: 수치 바인딩 밖 영역

✗ 시간 동결 (L2 JSON)

  • GitHub 스냅샷의 days_until·상대 문구는 생성 시각 기준
  • 소비자 미적용 배포가 다시 나오면 7/14 사고 재현
  • 배포 파이프라인 자동화 부재

✗ Track / 라이브 신호 희소

  • 스캔 창이 야간(KST)으로 한정
  • Stage2≥5 조건이 까다로워 신규 로그 드묾
  • “14일 미갱신” 헬스가 성능 문제와 혼동 가능

✗ 데이터 벤더 의존

  • yfinance rate-limit / 조정가 이슈
  • 실적 캘린더·EPS 컨센서스 지연·개정
  • 프리마켓·오버나이트 품질 변동

✗ 수집 스케줄 시차

  • brief / sentiment 슬롯 불일치 시 전일 심리 잔존 가능 (P1-1 잔여)
  • fetch 실패 시 stale-on-error 불완전 (서비스별)

✗ 제품 오용 리스크

  • Conviction “High”를 매수 신호로 오해
  • Polymarket 확률을 연준 가이던스로 오해
  • 소셜 optimistic을 추세 확정으로 오해 (일부 완화됨)

의도적으로 “약점”이 아닌 것

  • 백테스트 expectancy가 낮아 보이는 개별 종목(AMZN 등) — 방법론 한계 공개가 오히려 신뢰 요인
  • reference-only 예측시장 — 과신 방지 설계
  • 짧은 IPO(SPCX) 404 — 가짜 Stage2를 보여주지 않음

9. 결론

지금 SniperBoard는

감사 가능한 규칙 기반 시장 대시보드에 교정된 AI 컨텍스트를 얹은 형태다. 가격·레짐·실적 날짜·Stage2는 투자 참고의 1차 근거로 쓸 수 있는 수준(90대)에 진입했다.
